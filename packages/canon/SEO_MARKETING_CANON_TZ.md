## 1. РЕШЕНИЕ ЦИКЛА: pickup-only (и что с этим делать по каноне)

**Контекст.** Семантическое ядро построено на «только самовывоз». PRD и канон описывают **hybrid delivery** (самовывоз + курьер Яндекс Доставки). Подтверждено пользователем: **продукт сейчас pickup-only**, доставку убрать из интерфейсов.

**Что уже сделано в макетах (Claude Design):**
- Карточка букета / экран сделки: переключатель `DeliveryToggle` (Самовывоз/Курьер) → статичный блок **«Самовывоз рядом»** (`pd-scr-1.jsx`, `pd-scr-desktop.jsx`).
- Дизайн-система: демонстрация сегмент-контрола «доставка» → «свежесть» (`pd-ds.jsx`).
- Лендинг: «самовывоз **или доставка** по городу» → «самовывоз рядом»; футер «Доставка и самовывоз» → «Самовывоз рядом» (`pd-land.jsx`, `Тексты сайта.md`).

**Что должен сделать Claude Code / владельцы канона (ripple-list).** Не переписывать молча — это продуктовое решение, зафиксировать осознанно:

| Документ / артефакт | Сейчас | Действие под pickup-only |
|---|---|---|
| `DESIGN_BRIEF.md §4` | компонент `DeliveryToggle (самовывоз/курьер)` | пометить deprecated в MVP; оставить в коде за фиче-флагом `delivery.courier` (off) |
| `SCREEN_INDEX.md #5` | `DeliveryToggle/tracking` в экране сделки | заменить на «pickup-only: район/точка встречи в чате после договорённости» (обновлено в этом цикле — см. §7) |
| `API_CONTRACT.md` | `delivery_method: self_pickup\|courier`; `POST /deals/{id}/delivery`; FR-031 | бэк-контракт **оставить** (forward-compat), но фронт MVP всегда шлёт `self_pickup`; courier-ветки скрыты флагом |
| `FLOWS.md FLOW-6` | «Доставка курьером (трекинг) и фоллбэк» | пометить «отложено (рычаг роста); не в MVP» |

## 2. Семантическое ядро — резюме (полное: `peredarim-seo-yadro.md`)

**Стратегические выводы:**
1. Спрос на «передаренные букеты» не сформирован (бренд/категория ≈0) → строимся на перехвате «дешёвые/свежие цветы рядом».
2. Pickup-only отрезает «доставку цветов» (самый денежный кластер) → ставка на «рядом/у метро/самовывоз», объём меньше, конверсия честная.
3. Гео в Директе — таргетингом по регионам, **не** словами с городом. Город в ключах нужен **только** для SEO-страниц (Title/H1).
4. Инфо-кластеры (продавцу, доверие) некуда сажать без блога → рекомендован мини-блог 3–5 статей (дешёвый трафик на supply-сторону).

**Кластеры:** 1. Недорогие букеты · 2. Свежие цветы рядом/самовывоз · 3. Букет под повод · 4. Гео ×10 (только SEO) · 5. Бренд (вдолгую) · 6. Продавцу (пауза/блог) · 7. Доверие (пауза/блог).
**Приоритет:** 1+2 → 3 → бренд → 6–7 после блога.
**Частотности оценочные** — перед запуском прогнать через Wordstat по каждому из 10 регионов.

## 3. Маркетинговый / SEO-слой — что спроектировано

> Новый слой: публичные SEO-страницы поверх продуктового приложения. Макет — `Передарим · SEO-страницы.html` (design-canvas: гео ×10, безопасная сделка, блог, SEO-шаблон карточки). Десктоп + мобильный веб.

### 3.1 Гео-страницы городов ×10 — главный рычаг органики
Один шаблон, 10 URL. Город (со склонениями) и районы подставляются в Title/H1/тексты.
- **Структура:** хлебные крошки · eyebrow «Свежие букеты · {Город} · самовывоз» · H1 · вводный лид · 3 чипа доверия (дешевле/оплата при встрече/самовывоз) · разделитель · каталог с рабочими фильтрами (цена/свежесть), карточки `BouquetCard` · «букеты по районам {Города}» (перелинковка) · «недорогие букеты под повод» (чипы) · FAQ (почему дешевле/свежесть/как забрать/защита) · «свежие букеты в других городах» (перелинковка ×10).
- **Города:** Москва, СПб, Новосибирск, Екатеринбург, Казань, Нижний Новгород, Челябинск, Красноярск, Самара, Уфа.
- **Адаптив копи:** где нет метро (Челябинск, Красноярск, Уфа) — копи без «у метро» автоматически (флаг `metro` в данных города).

### 3.2 Страница «Безопасная сделка» — кластер доверия
Снимает главное возражение C2C («обман»). H1 «Платите за букет, только когда забрали его» · 3-шаговый флоу «оплата при встрече» (договорились → встреча → забирает самовывозом → деньги уходят) · плашка «если что-то не так» · текстовый разбор «как безопасно купить с рук» · FAQ · CTA. На неё ведёт футер и FAQ гео-страниц.

### 3.3 Мини-блог — прогрев supply (рекомендация, реализовано как макет)
Индекс из 3 статей + шаблон открытой статьи с CTA «Опубликовать букет». Темы из ядра: «Куда деть подаренный букет вместо мусорки», «Что делать с цветами после праздника», «Как продлить жизнь срезанному букету».

### 3.4 SEO-шаблон карточки букета
Заложить сразу, пока каталог наполняется: `Title {Цветы} за {цена} ₽ рядом, {район} — самовывоз | Передарим`; `alt свежий букет {цветы} недорого, {район}`.

## 4. Системные апдейты дизайн-системы (переиспользуемо во всём каноне)

> Это **токены и правила**, не одноразовая верстка. Должны попасть в `tokens/` и общий слой canon. Источник: `pd-seo.css`, `pd-typo.js`.

### 4.1 Типографика: неразрывные пробелы (анти-«висячие» предлоги)
Правила (см. `pd-typo.js`) — склеивать неразрывным пробелом (`\u00A0`):
- короткие слова (≤2 букв) и предлоги/союзы/частицы (`в, и, с, на, о, к, по, за, из, у, не, ни, же, бы, или, что, как, так, чтобы, если…`) с **следующим** словом/числом;
- числа и разряды/диапазоны (`2 490 ₽`, `2–3 раза`) — не отрывать от следующего слова/единицы;
- тире `—`/`–` и среднюю точку `·` — приклеивать к **предыдущему** слову (не начинают строку);
- **граница инлайн-узла:** короткое слово/число в конце текстового узла (перед `{интерполяцией}`, `<em>`, `<b>`) тоже склеивать — иначе «в» зависает на стыке (частый баг при шаблонах `в {город}`).
- Заголовки: `text-wrap: balance`; лиды/деки: `text-wrap: pretty`.

**Инженерное замечание (важно для SSR/SEO):** в Claude Design это runtime DOM-обходчик. В `web/` его нужно перенести на **build-time / string-трансформ** (или React-хелпер, вставляющий `\u00A0` в строку), чтобы краулеры видели неразрывные пробелы и не было layout-shift. Не тащить клиентский `MutationObserver` в прод как единственный механизм.

### 4.2 Токены отступов «заголовок → подзаголовок»
Системные переменные (на корне `.pds`, в каноне → семантические токены типографики):

| Токен | Назначение | Мобайл | Десктоп |
|---|---|---|---|
| `--pds-gap-eyebrow` | надзаголовок (eyebrow) → заголовок | 14px | 16px |
| `--pds-gap-deck` | заголовок → его подзаголовок/дек (тесная связка) | 10px | 12px |
| `--pds-gap-lede` | H1 героя → вводный лид-абзац | 22px | 30px |
| `--pds-gap-sechead` | блок заголовка → контент секции | 28px | 34px |
| `--pds-gap-qa` | вопрос FAQ → ответ | 9px | 10px |

**Грабли (зафиксировать в каноне):** глобальный ресет `.pd-root p, .pd-root h1…{margin:0}` имеет специфичность (0,1,1) и перебивает одноклассовые правила (0,1,0) на элементах `<p>`. Правила отступов **обязаны** быть специфичнее — скоупить под родителя (`.pds .pds-intro`, `.pds .pds-h2-sub` …) → (0,2,0). Иначе отступы молча схлопываются в 0.

### 4.3 Интринсик-сетка карточек (анти-overflow на мобайле)
Карточки никогда не сжимаются ниже комфортной ширины и не вылезают за экран; сетка **сама** убирает колонку.
```css
.pds-grid{ display:grid;
  grid-template-columns: repeat(auto-fill, minmax(min(100%, var(--pds-card-min)), 1fr));
  gap:14px; }
.pds-grid > *{ min-width:0; }   /* снимает min-content floor — дети не распирают колонку */
```
- `--pds-card-min: 240px` → узкий мобайл (≈360px контента) = **1 карточка в ряд**, десктоп (≈1068px) = **4 в ряд**, планшет = 2–3 (интринсик).
- **Грабли:** нативная сетка приложения (`.pd-grid`) работает 2-в-ряд на мобайле только из-за `.pd-grid>*{min-width:0}`. Любая новая card-grid обязана иметь `min-width:0` на детях, иначе контент-floor распирает контейнер за рамку телефона.
- Чипы (фильтры/доверие): `white-space:nowrap` — перенос целым чипом через `flex-wrap`, а не разрывом текста внутри пилюли.

| Route | Рендер | Примечание |

### 6.3 Данные городов (склонения — это ДАННЫЕ, не вычисление)
Для каждого города хранить формы и атрибуты:
```ts
type CityData = {
  id: string;            // slug: 'moskva'
  nom: string;           // 'Москва'        (именительный)
  loc: string;           // 'Москве'        (предложный — «в {loc}»)
  gen: string;           // 'Москвы'        (родительный — «районы {gen}»)
  count: number;         // витринный счётчик
  metro: boolean;        // есть ли метро (адаптив копи «у метро»)
  districts: { name: string; count: number }[];
};
```
Не генерировать падежи алгоритмически (особенно «Санкт-Петербург», «Нижний Новгород») — только справочник.

### 6.4 Definition of Done (приёмка, по `CANON_PACKAGE_TZ §12`)
- `web/` собирается чисто; `npm run test:visual` ≤ 2% против новых baseline (десктоп + мобайл, на каждый тип страницы; гео — Москва + один безметрошный город, напр. Челябинск).

- **Отступы:** computed `margin` для дек/лидов ≠ 0 (специфичность побеждает ресет §4.2).
- **Сетка:** мобайл — карточки не вылезают за вьюпорт (overflow-right = 0), узкий экран = 1 в ряд; десктоп = 4.
- **Мета:** Title/Description/H1/canonical/JSON-LD/sitemap соответствуют §5; склонения городов корректны.
- Версия `@rebloom/canon` (minor: новый entry `./marketing`) + секция в `CHANGELOG.md`; ASCII-маркеры новой сборки проверены (кириллица экранирована).
