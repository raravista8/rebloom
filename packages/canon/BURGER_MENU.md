# Мобильное меню (бургер) — логика для разработки

`@rebloom/canon` · `0.7.0` · компонент **`PdMobileMenu`** (экспорт из `./marketing`).
Источник правды: `src/marketing/landing.jsx` (design-source `pd-land.jsx`). Стили — `dist/canon.css` (`.pdl-drawer*`).

Бургер раскрывается в **правый sheet поверх затемнения**. Это меню **сайта**, поэтому контент один и тот же на лендинге и на SEO-страницах; различается только набор пунктов по состоянию авторизации.

---

## 1. API компонента

```jsx
import { PdMobileMenu } from "@rebloom/canon/marketing";

<PdMobileMenu
  open={menu}            // boolean — открыт ли sheet
  auth={false}           // boolean — гость (false) | авторизован (true)
  city="Москва"          // стартовый город (по умолчанию «Москва»)
  onClose={() => …}      // вызывается на ✕, тап по scrim, Esc и после тапа по любому пункту
/>
```

- Компонент **всегда отрисован** в DOM (не размонтируется): закрытое состояние — `visibility:hidden; pointer-events:none`, открытое — класс `.open`. Так анимация работает в обе стороны без таймеров размонтирования.
- Город — **внутренний state** компонента (инициализируется из `city`). Выбор города не навигирует, только меняет подпись и счётчик внутри шита. Проброс наружу (в общий стор гео) — на стороне `web/`.

---

## 2. Кто владеет состоянием

State `menu` живёт в **навигации**, не в самом меню:

```jsx
function Nav({ auth }) {
  const [menu, setMenu] = React.useState(false);
  return (
    <>
      <header className="pdl-nav">
        …
        <button className="pdl-nav-burger" aria-label="Меню"
                aria-expanded={menu} onClick={() => setMenu(true)}>
          <Menu />
        </button>
      </header>
      <PdMobileMenu open={menu} auth={auth} onClose={() => setMenu(false)} />
    </>
  );
}
```

**Критично:** `<PdMobileMenu>` монтируется **рядом с `<header>`**, а не внутри него. `.pdl-nav` — `position:sticky` (короткий по высоте); если положить drawer внутрь, `position:absolute; inset:0` отсчитается от хедера, а не от экрана. Сиблинг внутри `.pdl` отсчитывается от `.pdl`.

То же самое в `SeoNav` (`src/marketing/seo.jsx`) — общий `PdMobileMenu` импортируется из `./landing`.

---

## 3. Контейнмент (почему `absolute`, а не `fixed`)

- `.pdl` несёт `position:relative` (добавлено в 0.7.0).
- `.pdl-drawer { position:absolute; inset:0 }` → sheet покрывает ровно **рамку приложения** (`.pdl` = высота экрана; внутренний скролл живёт в `main.pd-scroll`), независимо от того, насколько прокручен документ.
- `position:fixed` здесь не годится: в кросс-платформенных/масштабируемых превью (девайс-фреймы, design-canvas с `transform`) fixed отсчитывается от layout-viewport и «вылезает» за рамку. На реальном `web/` оба варианта совпали бы, но `absolute` + `.pdl{position:relative}` корректен везде.

---

## 4. Открытие / закрытие / доступность

| Действие | Поведение |
|---|---|
| Тап по `.pdl-nav-burger` | `setMenu(true)` |
| Тап по `.pdl-drawer-x` (✕) | `onClose()` |
| Тап по `.pdl-drawer-scrim` | `onClose()` |
| Клавиша **Esc** | `onClose()` (слушатель вешается только пока `open`) |
| Тап по любому пункту/ссылке | сначала `onClose()`, затем навигация |

- На открытии фокус программно уходит на ✕ (`closeRef`, через `setTimeout` после появления).
- Бургер несёт `aria-expanded={menu}`; панель — `role="dialog" aria-modal="true" aria-label="Меню"`.
- Анимация (сдвиг панели + затемнение) отключается под `@media (prefers-reduced-motion:reduce)`.

---

## 5. Адаптив

- Бургер `.pdl-nav-burger` виден **только на мобильной ширине**; на `≥900px` (container-query по `.pdl`) — `display:none`, поэтому drawer на десктопе недоступен в принципе.
- Ширина панели: `min(88%, 360px)`. Тач-таргеты строк — `min-height:54px`.

## 6. Содержимое — ГОСТЬ (`auth=false`)

Порядок сверху вниз:

1. **Шапка:** знак + «Передарим» · ✕.
2. **Город** (`.pdl-drawer-city`) — строка «📍 Москва · N свежих букетов рядом»; тап разворачивает **inline-список** городов (radio, отмечен галкой). Без вложенных оверлеев.
3. **Навигация сайта** (`.pdl-drawer-nav` → `.pdl-drawer-row`):

| Пункт | Подпись | Назначение (`web/`) |
|---|---|---|
| Каталог букетов | Свежие букеты рядом | `/catalog` |
| Как это работает | Три простых шага | `/#how` (секция лендинга) |
| Безопасная сделка | Оплата при встрече | `/bezopasnaya-sdelka` (страница `PdSafeDeal`) |
| Приложение | iOS · Android · RuStore | `/#app` (секция лендинга) |

4. **Футер:** CTA `«Опубликовать букет»` (primary, во всю ширину) + строка «Уже с нами? **Войти**».

> В `reference/prototypes/*` ссылки — это `.html`-файлы превью (`Передарим · Каталог букетов.html`, `…Лендинг peredarim.ru.html#how`, `…SEO-страницы.html`). В `web/` маппинг — колонка «Назначение» выше. Якоря **абсолютные** (ведут на лендинг), потому что бургер открывается и на не-лендинговых страницах.

---

## 7. Содержимое — АВТОРИЗОВАН (`auth=true`)

1. **Шапка:** знак + «Передарим» · ✕.
2. **Профиль** (`.pdl-drawer-prof`) — аватар · имя · «Профиль и отзывы» · рейтинг ★. Тап → профиль пользователя.
3. **Город** — тот же inline-пикер, что у гостя.
4. **Разделы аккаунта:**

| Пункт | Бейдж | Назначение (`web/`) |
|---|---|---|
| Каталог букетов | — | `/catalog` |
| Избранное | `12` | `/favorites` |
| Мои букеты | — | `/my-listings` |
| Сделки и чат | `2` | `/deals` |
| Уведомления | • (точка) | `/notifications` |
| Настройки | — | `/settings` |

5. **Футер:** CTA `«Опубликовать букет»` + ghost-строка `«Выйти из аккаунта»`.

> Поиска в шите **нет** намеренно — на авторизованном экране строка поиска живёт в верхней панели (`.pdl-nav-search`), дублировать её в меню избыточно.

---

## 8. Гейтинг действий (гость)

Меню само по себе ничего не «защищает» — навигация по сайту открыта гостю. Стена входа возникает только на действиях, требующих аккаунта:

- **«Опубликовать букет»** (CTA в футере): гость → флоу `Вход/Регистрация` (`./auth`: `AuthChooser → OAuth | Phone→OTP → Register → Welcome`) → форма публикации (`./screens` · «Продать»). Если сессия уже авторизована — сразу форма.
- **«Войти»**: тот же auth-флоу; по завершении — возврат на текущую страницу, меню переключается на `auth`-вид.
- На карточке букета действия **«Написать продавцу»** / лайк аналогично упираются во вход, затем — чат сделки (`./deal`).

## 9. Чек-лист интеграции в `web/`

- [ ] Рендерить **один** `PdMobileMenu` на страницу, состояние `menu` держать в шапке; не плодить копии.
- [ ] Монтировать sheet **сиблингом** хедера внутри корневого `.pdl`; убедиться, что `.pdl` имеет `position:relative` (из `canon.css`).
- [ ] Прокинуть `auth` из сессии; для гостя/залогина — разные наборы пунктов отдаются компонентом автоматически.
- [ ] Заменить превью-`.html` ссылки на реальные роуты (§6/§7). Якоря лендинга — абсолютные (`/#how`, `/#app`), чтобы работали с любой страницы.
- [ ] Город из шита синхронизировать с общим гео-стором (сейчас — локальный state компонента).
- [ ] Бейджи (`Избранное`, `Сделки и чат`, точка «Уведомления») завести на реальные счётчики.
- [ ] Проверить блокировку фонового скролла при открытом шите, если потребуется (сейчас scrim перекрывает рамку; lock — на стороне `web/` при необходимости).
