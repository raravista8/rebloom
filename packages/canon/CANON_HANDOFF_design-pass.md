## 0. СНАЧАЛА ПРОЧЕСТЬ — три пункта меняют контракт/данные (параллельная backend-задача)

Дизайн нарисовал UI и заложил поля; бэкенд реализует в том же контракте.

1. **Метро.** Справочник станций по городам. В прототипе модель — `pd-feed.jsx`:
   - `PD_METRO_LINES` — `lineId → официальный цвет` (msk: `1..11`, spb: `sp1..sp5`).
   - `PD_METRO` — `cityKey → [{ n: 'Станция', l: ['lineId', …] }]` (пересадка = несколько lineId → несколько цветных кружков). Сейчас заполнены **msk + spb** (центральные станции).
   - `PD_CITY_METRO` — `cityName → cityKey` (города, где есть метро). Города без метро → fallback на район.
   - `PD_METRO_INDEX` / `pdMetroLines(station)` — станция → линии.
   - **→ API_CONTRACT:** `metro` в `listing_card`/`listing`; `metro` в `POST /listings`; `metro[]` в `/feed`+`/search` (OR — «любая из выбранных»); сид станций/цветов msk+spb (далее остальные города).
2. **Типы цветов.** Фиксированный список — `PD_FLOWERS` (полный, для формы) и `PD_FLOWER_FILTERS` (частые, для фильтров) в `pd-feed.jsx`. **→ API_CONTRACT:** канонический enum/справочник; `flower_types[]` у объявления; `POST /listings`; фильтр `flower_types[]` (OR).
3. **«Показать N букетов»** в мобильной панели фильтров — нужен счётчик результатов под текущие фильтры **до** применения. Прототип считает по уже загруженному списку. **→** клиентский счёт или лёгкий count-эндпоинт. [verify]

**CHANGELOG-пометка для экспорта:** новые data-поля `metro`, `flower_types[]`; фиксированный справочник станций+цветов.

## 1. Метро (новый ориентир)
- **Было:** только район («Москва · Патрики»). **Стало:** город + станция метро; в городах с метро станция — главный ориентир. Города без метро → fallback на район (Card: `d.metro ? <MetroLabel/> : район`).
- Реальные станции Москвы и СПб с официальными цветами линий; пересадки (Киевская, Курская, Парк культуры, Комсомольская…) — несколько кружков. Компонент: `PdMetroLabel` / `PdMetroDots` (`pd-feed.jsx`), стили `.pd-metro`, `.pd-mdots` (`pd.css`).
- **Публикация:** «Район» (текст) → `PdMetroPicker` (поиск + список с цветами линий), `pd-kit.jsx`.
- **Фильтр по метро** (каталог: сайдбар + моб. панель; лендинг): **мультивыбор** — `PdMetroPicker multi values onToggle`; чекбоксы, теги выбранных станций с крестиком (`.pdc-metrotag`), логика OR. Кнопка «Готово» / «Сбросить (N)».

## 2. Фильтры (главная + каталог)
- **Было:** цена, свежесть, рейтинг, размер. **Стало:** + **тип цветов** + **метро**.
- Типы цветов (OR), список: Розы · Пионовидные розы · Пионы · Тюльпаны · Гортензия · Хризантемы · Эустома · Ранункулюсы · Альстромерия · Лилии · Полевые.
- **Мобайл:** чип «Фильтры» раньше ничего не открывал → теперь раскрывает inline-панель **«Все фильтры»** (`.pdc-panel`) со всеми группами + «Сбросить» / «Показать N букетов». (Inline-панель, а не bottom-sheet: фрейм каталога — страница во всю высоту без фикс-вьюпорта, overlay уезжал бы в самый низ. В вендоринге с фикс-вьюпортом можно вернуть sheet.)
- **Сортировка:** моб. лента кнопок (обрезалась) → дропдаун `SortDropdown` (`.pdc-sortdd`); десктоп — сегментный (`.pdc-sortsel`).
- Размеры в фильтре подписаны: S · до 7 · M · 7–15 · L · 15–25 · XL · 25+.

## 3. Карточки букетов (главная, каталог) — `PdCard` в `pd-feed.jsx`
- «L» → «Размер L», сделан **тихим**: приглушённый текст у цены (`.pd-size`), без плашки.
- Город/район убран → станция метро.
- Рейтинг продавца → вплотную к имени (`.pd-rating`, `.pd-seller`).
- Длинное имя продавца → ellipsis, рейтинг остаётся (`.pd-seller-n{overflow:hidden…}`, `.pd-rating{flex:none}`).

## 4. Бейджи свежести — `PD_FRESH_META` (`pd-feed.jsx`)
- На карточках: **«Свежий»** (вместо «сегодня») + короткие **«1–2 дня»**, **«3+ дня»**. В фильтре добавлен «3+ дня».
- **Enum не меняется:** `today | d1_2 | d3_plus` — поменялись только подписи (`label`).

## 5. Страница букета — `pd-scr-1.jsx` (моб.) + `pd-scr-desktop.jsx` (десктоп)
- Шеринг → тихая **icon-кнопка** «Поделиться» (`.pd-sharebtn`), без текста, не тянет внимание.
- Цена убрана с кнопок CTA.
- Локация: город + станция метро; «Как забрать» = «Самовывоз у м. <станция>».
- Убрана подпись «Самовывоз. Точное место… двор/станцию выбирает продавец».
- Продавец: блок «Продавец» → «Продаёт <Имя>», подпись «рейтинг продавца», число сделок — отдельной статой (`.pd-sellercard` → `.pd-seller-deals`, не путается с рейтингом).
- Размер/состав/локация: единый шрифт/начертание, ровные строки (`.pd-buy-meta`, `.pd-buy-spec`, `.pd-buy-loc`).
- Состав: просто текст через «·» (`.pd-flowerlist`), не кнопки, без цветных точек.

## 6. Форма добавления букета — `pd-scr-2.jsx` (моб.) + `pd-scr-desktop.jsx` (десктоп)
- «Район» (текст) → `PdMetroPicker` (поиск).
- Добавлен необязательный мультивыбор типов цветов — `PdFlowerPicker` (список §2).
- Убран хинт «Похожие букеты… 700–1 300 ₽».
- «Свежесть: Сегодня…» → «Когда букет подарили».

## 7. Шапка (десктоп + главная + каталог)
- Старый `pdw-nav` (плоский, «Сделки», CTA «Продать букет») → **актуальная шапка как на лендинге** (`pdl-nav`): липкая с блюром, белые скруглённые контролы, круглый аватар, «Избранное» (сердце `PdHeart`), CTA «Опубликовать букет»; иконки одного размера.
- **Каталог:** desktop — полный набор (бренд · город · поиск · уведомления · избранное · аватар · CTA); **мобайл — бренд + уведомления + аватар + бургер**, бургер открывает выезжающее меню-драйвер (`.pdl-drawer`: профиль, выбор города, ссылки, CTA внизу) — те же классы, что на лендинге. Источник правды шапки/драйвера = `pd-land.jsx` (`pdl-nav`/`pdl-drawer` в `pd-land.css`).
- **Заметка потребителю (Claude Code):** app-шапка в `web/` — `WebChrome` (хэндролл, auth-aware). При вендоринге портировать canon-эталон в `WebChrome`, сохранив `useMe()`: гость не видит «Избранное»/аватар/«Сделки», видит «Войти».
- **Известное ограничение прототипа:** в `Клиентские экраны` / `Главная` десктоп-шапка использует `pdw-nav` (визуально приведён к canon в `pd.css`), но без мобильного бургер-драйвера (там нет mobile-web-фрейма; мобильные экраны — нативные app-bar `PdScreen`/`PdTopBar`). Единый бургер-драйвер реально нужен только на mobile-web — лендинг и каталог. Если нужно физически единый JS-модуль шапки — вынести `PdWebNav` в общий файл (отдельная задача, см. §11).

## 8. Сделка / профиль / вёрстка
- «Букет M · Патрики» → «Букет M» (район не сливается с названием) — `pd-scr-3.jsx`, `pd-scr-desktop.jsx`.
- «с 2025 на площадке» → «10 месяцев на площадке» (относительный срок).
- Десктопные витрины: артборды под полную высоту (HTML `deskTall`), не обрезаются.
- Мобильный веб: убран notch-отступ у шапки экрана (`.mbw-view .pd-appbar` в `pd-auth.css`).
