# HANDOFF 0.5.0 — «Оплата при встрече» (убран эскроу) · для Claude Code

> **Что произошло.** Запускаемся **без эскроу-сервиса и без онлайн-оплаты**. Площадка больше не проводит, не держит и не возвращает деньги — люди договариваются в чате и платят **при встрече** (наличные/перевод), как на доске объявлений. Это затронуло **модель сделки, API, экраны, админку, SEO и все ТЗ**. Документ — полный список изменений канона `@rebloom/canon` 0.4.0 → **0.5.0** и чек-лист для бэкенда/веба.

---

## 0. TL;DR

- Платёжный слой (эскроу, эквайринг, выплаты, комиссия, возвраты, чарджбэки, вебхук ЮKassa) **удалён из продукта**. Монетизация (платные публикации/продвижение/комиссия) — **отдельный ADR post-MVP**.
- **Новая стейт-машина сделки:** `agreed → meeting → done` (+ `problem`, `cancelled`). Денег платформа не касается.
- **Доверие** строится на: оплата при встрече (нет предоплаты) · самовывоз рядом · реальные отзывы и рейтинг · верифицированный вход · модерация · жалобы.
- **Споры → жалобы.** Поддержка/модерация не двигает деньги, а предупреждает/ограничивает/блокирует сторону или снимает объявление.
- Экран **«Оплата не прошла» удалён**. Раздел настроек «Платежи и выплаты» → «реквизиты для перевода». Админ-«Финансы» → аналитика сделок (без ledger/комиссии).

## 1. Модель сделки (было → стало)

| | Было (эскроу) | Стало (оплата при встрече) |
|---|---|---|
| Статусы | `created → paid_held → released \| refunded \| disputed \| cancelled` | `agreed → meeting → done \| problem \| cancelled` |
| Деньги | эскроу на номинальном счёте, релиз минус комиссия | **не проходят через платформу**; расчёт P2P при встрече |
| Старт сделки | «Купить · N ₽» → оплата картой/СБП | **«Написать продавцу»** → создаётся `deal:agreed`, открывается чат |
| Адрес | раскрывается после оплаты | раскрывается **после договорённости** (`meeting`, `share-point`) |
| Завершение | «Подтвердить получение» → релиз денег | «Подтвердить получение» → `done`, открываются отзывы |
| Конфликт | спор, заморозка денег, релиз/возврат | **жалоба** (`problem`) → модерация: предупредить/ограничить/заблокировать/снять объявление |
| Отмена | возврат по правилам эквайринга | `cancelled`, объявление → `active`, деньги не задействованы |

**Маппинг статусов для миграции данных:** `created→agreed`, `paid_held→meeting`, `released→done`, `disputed→problem`, `refunded→cancelled`, `cancelled→cancelled`.

## 2. API-контракт (`API_CONTRACT.md`) — дельта

- enum `deal.status` = `agreed | meeting | done | problem | cancelled`.
- `delivery_method` = `self_pickup` (courier — post-MVP, без участия платформы в деньгах).
- **Удалено:** `payment` в ответе `POST /api/deals`; `POST /api/webhooks/yookassa`; `confirm-receipt`→release-семантика; ошибка `payment_failed`; поля `amount_kopecks`/`commission_kopecks`/`released_at`; `account/payment-methods`·`payout-method`.
- **Добавлено/изменено:**
  - `POST /api/deals` `{listing_id, delivery_method}` → `{deal}` (status `agreed`, открыт чат).
  - `POST /api/deals/{id}/share-point` (seller) → `meeting`, адрес виден покупателю.
  - `POST /api/deals/{id}/confirm-receipt` (buyer) → `done`.
  - `POST /api/deals/{id}/report` `{reason, photo_ids[]}` → `problem` (вместо `/dispute`).
  - `POST /api/deals/{id}/cancel` `{reason?}` → `cancelled`, листинг → `active`.
  - `POST /api/account/transfer-details` `{card_last4?, phone?}` — реквизиты для прямого перевода.
- `deal`: `{id, status, listing, role, counterparty, delivery_method, pickup_point?, created_at, done_at?}`. `price_kopecks` — справочная цена объявления, **не** платёж.
- Админ: `/api/admin/deals` фильтр по новым статусам; `POST /api/admin/deals/{id}/cancel` (вместо `/stop` и `DELETE …/refund`); `/api/admin/finance` → оборот сделок (оценка по `done`) + сделки по статусам, без ledger/комиссии; `/api/admin/overview` KPI `deal_turnover_kopecks` + `deals_by_status{}` вместо `gmv/commission`.

## 3. Переименования (важно для импортов и БД)

- **Компоненты экрана сделки** (`./deal`, `./screens`):
  - `DealPaidHeld` → **`DealActive`** (активная сделка, статус `meeting`)
  - `DealDisputed` → **`DealProblem`**
  - `DealReleased` → **`DealDone`**
  - `PaymentFailed` — **в 0.5.0 это заглушка-плейсхолдер** (нейтральный текст «оплата при встрече»); **удалить из `entries/deal.jsx` и `screens/deal-notifications` при следующей сборке** — экран не нужен.
- **Степпер** `PdStepper` (`./primitives`): `DEAL_STEPS = [agreed «Договорились», meeting «Встреча», done «Завершено»]`; `status ∈ agreed|meeting|done|problem`.
- **Админ-ключи статусов** во всех таблицах/бейджах приведены к `agreed/meeting/done/problem/cancelled`. CSS-бейджи: `.pda-badge.{done|meeting|problem|agreed|cancelled}` (добавлены в `canon.css`; старые `.held/.released/.disputed` оставлены, не используются — можно удалить).
