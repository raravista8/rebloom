# SUPPORT & DISPUTES — Передарим (`rebloom`)

> Поддержка пользователей и разрешение споров по сделкам. Роль `support` (RBAC уже есть). Связано с deals (FR-024), admin, AUDIT_LOG.

## 1. Каналы поддержки
In-app форма/чат поддержки + email. Категории: сделка/оплата/доставка/модерация/аккаунт/другое.

## 2. Споры по сделке (escrow)
- Открытие: любая сторона до релиза → сделка `disputed`, **деньги заморожены** (fail-secure).
- Доказательства: фото/описание от сторон в кейсе.
- Решение `support` (в лимите суммы) / `admin` (выше лимита, 4-eyes): `released` / `refunded` / частичный возврат — всё в ledger + audit.
- SLA (цель): первичный ответ ≤ 24ч; решение спора ≤ 72ч `[verify]`. Эскалация support→admin при превышении/высокой сумме.

> Пошаговый UI спора (сторона пользователя и саппорта) — `handoff/FLOWS.md` FLOW-1.

## 3. Консоль саппорта (в админке)
Очередь обращений и споров с приоритетом и SLA-таймерами; карточка сделки (участники, платёж, ledger, чат, доставка); действия по роли (с причиной + audit). Состояния — по INTERACTION_STATES.

## 4. Задачи
Epic E12 (T12.5 support-консоль + dispute SLA/эскалация). Базовый dispute-флоу — T5.6.
