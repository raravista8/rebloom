# NOTIFICATIONS — Передарим (`rebloom`)

> ⚠️ **ОБНОВЛЕНО — ADR-0013 (запуск без эскроу).** Триггеры статусов сделки: `agreed` (новый покупатель → продавцу), `meeting` (продавец назначил встречу → покупателю), `done` (завершено → обе стороны), `problem` (жалоба → второй стороне), `cancelled`. Уведомления про оплату/выплату/эскроу/возврат удалены — денег площадка не двигает.

> Многоканальные уведомления + админ-рассылки + баннер техработ. Модуль `core/notifications`. Каналы: **push** (mobile, Capacitor), **email** (Yandex SMTP 465), **Telegram** (через прокси, OPERATIONS §4), in-app.

## 1. Принципы
- Событие → правило → канал(ы) по приоритету и согласиям пользователя; best-effort, идемпотентно (одно событие — не больше одного уведомления на канал).
- Критичное по деньгам/сделке шлём надёжными каналами (push+email), маркетинг — только с согласием (opt-in), с отпиской.
- PII в теле — минимально; ссылки — на экран, не данные в URL (SECURITY privacy).

## 2. Триггеры (транзакционные, FR-050)
Регистрация/вход (OTP — только SMS, см. auth), новое сообщение в сделке, смена статуса сделки (paid_held/released/disputed/refunded), новая покупка у продавца, объявление прошло/не прошло модерацию, спор открыт/решён, выплата отправлена, отзыв получен/доступен, блокировка/предупреждение.

## 3. Каналы и шаблоны
- **push**: заголовок + короткий текст + deeplink на экран; тихие для второстепенных.
- **email**: брендированный шаблон «Передарим», тема + тело RU, ссылка-CTA, футер с отпиской (для маркетинга) и реквизитами; SMTP Yandex (app-password).
- **Telegram**: сообщение в бота (egress через прокси); только для связанных TG-аккаунтов.
- Шаблоны версионируются, тексты — RU, тон из DESIGN_BRIEF.

## 4. Настройки пользователя
Категории: «сделки» (всегда вкл, нельзя отключить критичные), «сообщения», «маркетинг/новости» (по умолчанию выкл). Per-channel toggles. Отписка от email в один клик.

## 5. Админ-рассылки и баннер
- `admin`-роль: массовая рассылка (сегмент: город/платформа/статус) с предпросмотром, лимитом и **подтверждением + audit**; только в разрешённые категории (маркетинг — по согласию).
- **Баннер техработ/инцидента**: флаг в админке → in-app/website баннер (текст + период); state в `INTERACTION_STATES` (global banner).

## 6. Бэкенд
- `core/notifications`: outbox-таблица + воркер (RQ), ретраи с backoff, дедуп по (event_id, channel).
- Провайдеры за портами: `PushProvider`, `EmailProvider`, `TelegramProvider` (адаптеры в `infrastructure/`).
- Метрики доставки; маркетинг-согласие в `consent`.

## 7. Задачи
Epic E12 (T12.1 транзакционные + outbox; T12.2 настройки/отписка; T12.3 админ-рассылки + баннер).
