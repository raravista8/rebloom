# ADR-0010: Platform commission model (рабочий дефолт)
Date: 2026-06-03
Status: Accepted (default; меняется параметром, оператор может пересмотреть)

## Context
Нужна явная модель комиссии, чтобы достроить эскроу/сплит (E5) и финчасть админки (E11). Оператор не зафиксировал значение; ставим дефолт, разблокирующий разработку.

## Decision
- **Покупатель платит цену объявления** (ровно, без скрытых надбавок). Курьер (если выбран) — поверх, платит покупатель.
- **Комиссия площадки — по умолчанию 10% с ПРОДАВЦА**, удерживается из выплаты через сплит ЮKassa (`PLATFORM_COMMISSION_BPS=1000`).
- **Эквайринг/эскроу-комиссия ЮKassa** (СБП дешевле; «Безопасная сделка» до ~4.5%) — **за счёт площадки** из её комиссии (UX-простота: покупатель видит одну цену).
- Значение и сторона-плательщик — конфигурируемы (`PLATFORM_COMMISSION_BPS`, флаг who_pays) и пересматриваются по юнит-экономике.

## Alternatives
- Комиссия с покупателя (надбавка к цене) — хуже воспринимается, но защищает маржу; оставлено как переключатель.
- Сплит 50/50 — усложняет коммуникацию цены; отклонено для дефолта.

## Consequences
Positive: понятная цена для покупателя, простой сплит, финчасть считается из ledger.
Negative: при низком чеке 10% может не покрывать эквайринг — мониторить минимальную цену/чек; возможен мин. флор комиссии.

## Verification
`deal.commission_kopecks` = 10% от цены (или конфиг); ledger сходится (commission = доход площадки − эквайринг); E11 finance показывает оборот и комиссию.
