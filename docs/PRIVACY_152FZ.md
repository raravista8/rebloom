# PRIVACY & ФЗ-152 — права субъекта ПДн и retention · Передарим (`rebloom`)

> Хранение ПДн — в РФ (ФЗ-152). Этот документ: согласия, права субъекта (экспорт/удаление), retention. Связано с SECURITY (T-10/T-16), ADMIN (доступ логируется).

## 1. Согласия
- При первом ПДн-действии — версионированное согласие (FR-004): `policy_version`, `accepted_at`, `source_channel`. Маркетинг — отдельное opt-in согласие.
- История согласий хранится; изменение политики → повторное согласие.

## 2. Права субъекта (DSR)
- **Экспорт данных**: по запросу пользователь получает свои данные (профиль, объявления, сделки, отзывы, сообщения) машиночитаемо — задача с проверкой личности (через сессию/OTP).
- **Удаление аккаунта**: запрос → soft-disable немедленно, затем удаление/обезличивание ПДн в срок (см. retention), **с сохранением** того, что обязаны хранить по закону/для финучёта (обезличенные ledger-записи денежных операций).
- **Исправление**: пользователь правит свои данные; админ-правка — только с причиной + audit (T-16).
- Запросы DSR логируются и обрабатываются в SLA.

## 3. Retention (сроки хранения)
| Данные | Срок | Примечание |
|---|---|---|
| Профиль (телефон, имя) | пока активен + N после удаления | затем обезличивание |
| Фото объявлений | до архивации + grace | удаление из Object Storage по lifecycle |
| Сообщения чата сделки | срок сделки + спорное окно | затем обезличивание |
| IP / device / session | ограниченный срок (антифрод) `[verify юр-срок]` | минимально необходимый |
| Финансовые/ledger | по закону (бухг./54-ФЗ) | обезличенно, не удаляется по DSR |
| Audit log | 1 год | неизменяемый |
> Точные сроки `[verify с юристом]`.

## 4. Контроли (cross-ref SECURITY)
- Шифрование чувствительных колонок (адреса/гео) AES-256-GCM; managed-DB at-rest; least-priv роли.
- Доступ к ПДн/IP — role-gated + audited; `analyst` — только агрегаты.
- Никаких ПДн в логах/URL/в LLM без маскирования.
