# ADMIN — DESIGN BRIEF (ТЗ для Claude Design) · Передарим (`rebloom`)

> Дизайн **админ-панели** оператора. Десктоп-first, data-dense. Использует ту же дизайн-систему `@rebloom/canon` (+ admin-подмножество data-компонентов). Парное ТЗ по бекенду — `ADMIN_BACKEND_TZ.md`. Состояния — по `INTERACTION_STATES.md`, моушн — по `MOTION.md` (в админке сдержанно). Данные/эндпоинты — по `ADMIN_BACKEND_TZ.md §API`.

## 0. Платформа и доступ
- **Desktop-first** web (плотные таблицы, графики, drill-down). Адаптив до планшета; телефон — только просмотр ключевых KPI.
- Вход — только для ролей `admin` / `moderator` / `support` / `analyst` с **2FA (TOTP)**. Разные роли видят разный объём (см. `ADMIN_BACKEND_TZ.md §RBAC`). Дизайн предусматривает скрытие/disable действий вне роли.
- **Чувствительные данные (IP, ПДн, редактирование юзера)** визуально помечаются как «доступ логируется» (ФЗ-152).

## 1. Экраны
1. **Dashboard (обзор)** — KPI-карточки: **online сейчас**, **DAU / MAU**, всего юзеров, юзеры по городам, юзеры по платформам (web/iOS/Android), **GMV (оборот)**, **комиссия за период**, число сделок по статусам. Графики: прирост юзеров по времени (с разбивкой по городу и **источнику/UTM**), темпы прироста, оборот/комиссия по периодам. Переключатель периода (день/неделя/месяц/произвольный).
2. **Пользователи** — таблица с **поиском (по имени/телефону/id)** и фильтрами (город, статус, платформа, риск). Колонки: имя, город, платформа, кол-во объявлений, рейтинг, кол-во отзывов, последний IP, **флаг «несколько аккаунтов с этим IP»**, статус, дата регистрации. Сортировка. Bulk-действия. 
   - **Drill-down пользователя**: профиль, его объявления, отзывы на него + рейтинг, его сделки, список IP/устройств + платформы, риск-сигналы; действия: **блокировать/разблокировать**, **редактировать данные** (с подтверждением + причиной), сбросить сессии.
3. **Объявления** — таблица + **поиск по объявлениям**, фильтры (город/статус/свежесть/модерация), очередь `pending_review` с approve/reject (+причина). Drill-down: фото, продавец, история.
4. **Сделки** — таблица с фильтром по статусам (created/paid_held/released/disputed/refunded/cancelled), суммы, комиссия. Действия: **остановить сделку** (заморозить), **удалить/отменить** (soft, с возвратом по правилам) — оба destructive: модалка-подтверждение + причина (+ 4-eyes для денежных по `ADMIN_BACKEND_TZ.md`). Drill-down: участники, платёж, ledger, чат, спор. **Экран разрешения спора (release/refund/partial/escalate, confirm+reason, 4-eyes) — `FLOWS.md` FLOW-1 шаг 4.**
5. **Финансы** — **общий оборот проекта**, **заработок на комиссии за периоды**, выплаты, заморозки/споры, расхождения ledger (сверка). Экспорт за период.
6. **Антифрод / подозрительная активность** — лента сигналов и кластеров: мульти-аккаунты по IP/устройству, self-dealing (накрутка отзывов), аномалии цен, переиспользование фото, концентрация выплат на одну карту, всплески споров и т.д. (методики — `ADMIN_BACKEND_TZ.md §Fraud`). По каждому: риск-score, действия (в очередь, ограничить, заблокировать).
7. **Модерация** — единая очередь текст/фото кейсов (incl. мат/слуры из `MODERATION.md`), апелляции.
8. **Жалобы пользователей** — входящие репорты на объявления/юзеров.

## 2. Компоненты (admin-набор поверх canon)
`DataTable` (сортировка/пагинация/фильтры/выбор строк/sticky header, поиск), `KpiCard` (значение + дельта + спарклайн), `Chart` (линия/бар/донат — прирост, оборот, разбивки), `PeriodPicker`, `FilterBar`, `DrillPanel` (боковая панель/страница сущности), `StatusBadge` (статусы сделок/юзеров/модерации), `ConfirmDestructiveModal` (с полем «причина»), `AuditTrail` (история действий), `RiskBadge`, `IpCluster` (визуализация связки аккаунтов по IP), `Money`, `EmptyState`, `Toast`.

## 3. Состояния (по `INTERACTION_STATES.md`)
Каждая таблица/график: loading (skeleton), loaded, **empty** (нет данных), **no-results** (поиск/фильтр 0), error, offline. Destructive-действия: idle → confirm (модалка + причина) → in-flight (loading, кнопка активна-но-в-работе) → success/error. Кнопки активны по умолчанию.

## 4. UX-правила админки
- Плотность данных без перегруза: разумные default-колонки + «ещё колонки». 
- Любое destructive/деньги/PII-действие = подтверждение + обязательная **причина** (уходит в audit).
- Числа крупно и читаемо (деньги — из копеек в ₽, разделители разрядов, период явно).
- Графики прироста: абсолют + темп (%/период), разбивки переключаются (город/платформа/источник).
