# ТЗ для Claude Design — экран «Каталог букетов» (`/catalog`)

> Заполненный **§1 «ЧТО СДЕЛАТЬ»** универсального промта `CLAUDE_DESIGN_PROMPT.md` — §0 и §2–§6
> применяются без изменений (mobile-first, токены, все состояния, a11y, no-escrow, контракт
> упаковки, baseline). Парные: `DESIGN_BRIEF.md`, `INTERACTION_STATES.md`, `API_CONTRACT.md`,
> `SCREEN_INDEX.md`.
>
> **Зачем.** В экспорте нет отдельной страницы каталога. Кнопка «Весь каталог букетов» (лендинг) и
> пункт меню «Каталог букетов» ведут на `/catalog`, но такой страницы нет — `/search` это
> **поиск-first** (в idle — пустая подсказка «введите запрос»), а не каталог. Нужен полноценный
> **browse-first** экран: сетка всех букетов города сразу, без гейта на ввод запроса.

## Экран
**`PdCatalog`** — «Каталог букетов», публичный роут `/catalog` (гость может листать, как и главную).
Экспортировать из canon (entry `./feed` или `./catalog`), чтобы `web/` импортировал компонент +
доехал CSS. Переиспользовать те же грид-классы/карточку, что лендинг и поиск (`.pdw-grid`/`.pd-grid`,
`BouquetCard`/`.pd-card`), — консистентность и авто-пропагация стилей.

## Главное действие / суть
Полный **browse** каталог города: крупная сетка `BouquetCard` (фото-каррусель, `FreshnessBadge`,
`LikeButton` со счётчиком, цена, город, `SellerBadge`) — показывается **сразу**, без «введите запрос».

## Элементы
- **Шапка/навигация:** общий app-хедер (на десктопе — `WebChrome`-эквивалент, auth-aware: гость видит
  «Войти»; на мобайле — app-bar). H1 «Каталог букетов».
- **Город:** селектор текущего города (как в хедере); каталог скоупится по `city_id`.
- **Сортировка/секции:** переключатель «Самые свежие» / «Самые залайканные» (`feed.section = fresh | liked`).
- **Фильтры:** размер (S/M/L/XL), свежесть (`today | d1_2 | d3_plus`), диапазон цены. Кнопки/чипы как в поиске.
- **Вход в поиск:** опциональная строка/иконка «Поиск» → ведёт на `/search` (поиск-first остаётся отдельно).
- **Пагинация:** «Показать ещё» (курсор) или бесконечный скролл; карточки появляются каскадом (stagger, по `MOTION.md`).

## Данные (по `API_CONTRACT.md`)
`GET /api/feed?city_id&section=fresh|liked&cursor&limit` → `{ items: listing_card[], next_cursor, applied:{city_id,filters} }`
для базового каталога; при заданных фильтрах — `GET /api/search?city_id&size?&freshness?&price_min?&price_max?&cursor&limit`.
`end-of-list` = `next_cursor: null`. Деньги — копейки (`price_kopecks` → ₽). Никаких выдуманных полей.

## Состояния (обязательно все — `INTERACTION_STATES.md §4/§6`)
`loading` (skeleton-сетка, не спиннер) · `loaded` · **`empty`** (в городе ещё нет объявлений → дружелюбный
empty-state + CTA «Будьте первым — опубликуйте букет» / сменить город) · **`no-results`** (фильтр дал 0 →
отдельный текст + «сбросить фильтры», ≠ empty) · `loading-more` (нижний спиннер) · `end-of-list`
(стоп-маркер) · `error` (текст + «Повторить») · `offline` (баннер). Кнопки активны по умолчанию.

## Отличие от `/search`
Каталог — **browse-first** (сетка сразу, фильтры опциональны). Поиск (`/search`) — **query-first** (в idle —
подсказка ввести запрос). Это два разных экрана; «Каталог букетов» НЕ должен открывать поиск-экран.

## Копирайт / правила
RU, тёплый тон; «Каталог букетов», «Показать ещё», «Самые свежие»/«Самые залайканные». **No-escrow**
(ADR-0013): никакой «эскроу/предоплаты/комиссии». Без завершающих точек в заголовках.
