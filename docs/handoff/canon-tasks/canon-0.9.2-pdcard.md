# ТЗ для Claude Design — canon 0.9.2: `PdCard` data-driven (блокер импорта `PdCatalog`)

> 0.9.1 сделал `PdCatalog` презентационным — спасибо. Но **импортировать его на живой `/catalog`
> всё ещё нельзя**, потому что его карточка `PdCard` (`src/feed/feed.jsx`) не data-driven по двум
> местам — и web вынужден держать хэндролл `CatalogScreen` (свой `BouquetCard` с настоящим фото и
> рабочим лайком). Чиним карточку — тогда web удалит хэндролл и импортит `PdCatalog`.

## Блокеры (обязательно)
1. **Фото — настоящий URL.** Сейчас `feed.jsx:183` рендерит `src={PD_IMG(d.photo)}`, где
   `PD_IMG = id => `img/${id}.jpg`` — то есть всегда локальный Unsplash-id. У нас фото приходит
   **полным URL** (`photo_thumb_url`, CDN/Object Storage). → Карточка должна принимать **полный URL**
   (отдельное поле `photoUrl`, или детект `String(photo).startsWith('http')` → использовать как есть,
   иначе `PD_IMG`). Иначе на проде все карточки каталога — битые картинки.
2. **Лайк — наружу.** Сейчас `LikeBtn` чисто оптимистичный (локальный `useState`, без API). У нас лайк
   персистится (`POST /api/listings/{id}/like`, гость → `/login`). → Карточка должна принимать
   `onLike(id, next)` (и `liked`/`likes` как controlled-пропы), чтобы web подключил реальный лайк.
   Альтернатива: **`PdCatalog`/`PdCard` принимает card render-prop** (`renderCard={(item)=>…}`) — тогда
   web подставит свой `BouquetCard` целиком, и пункты 1–2 решаются разом. Любой из вариантов годится.

## Желательно (мелочи, всплыли при импорте)
3. **Новый продавец.** `PdCard` зовёт `seller.r.toFixed(1)` → при `null` рейтинге рисует «0.0».
   Должно быть «Новый» (как в нашем хэндролле) при отсутствии рейтинга/сделок.
4. **Сортировка.** `PdCatalog` эмитит `sort ∈ {fresh, cheap, exp, rating}`, а у бэкенд-ленты только
   секции `fresh`/`liked` (нет сортировки по цене). → Либо `PdCatalog` сортирует `cheap`/`exp`
   клиентски по уже выданным `items`, либо убрать неподдерживаемые опции (оставить `fresh`/`liked`).
5. **Рейтинг-фильтр.** В сайдбаре/панели есть фильтр по рейтингу продавца — бэкенд его не поддерживает
   (out of scope). Пока он inert. Если оставляете — ок; если хотите, чтобы работал — нужен бэкенд-фильтр
   (отдельная задача).

## Приёмка
`PdCard` показывает картинку из полного URL и поднимает лайк наружу (или есть `renderCard`); web
удаляет `components/catalog/CatalogScreen.tsx`-хэндролл и рендерит `<PdCatalog items=… onLike=…
renderCard?=… />`; фото и лайки на `/catalog` работают как на главной; визуал ≤2%. Версия → 0.9.2,
секция в `CHANGELOG`, дифф `src/` показывает `feed.jsx` (+`catalog.jsx` если добавили `renderCard`).
