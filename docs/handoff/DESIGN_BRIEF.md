# DESIGN BRIEF — ТЗ для Claude Design · Передарим (`rebloom`)

> Это техзадание для проектирования UI в **Claude Design**. Парный документ — `CANON_PACKAGE_TZ.md` (как упаковать результат в npm-пакет `@rebloom/canon` и передать в Claude Code). Источник продукта — `../PRD.md`, экраны — `SCREEN_INDEX.md`, формы данных — `API_CONTRACT.md`, **состояния — `INTERACTION_STATES.md`**, **пошаговые сложные флоу (спор и т.д.) — `FLOWS.md`**, **ховеры/анимации — `MOTION.md`** (всё обязательно). Дизайн **админки** — отдельно в `ADMIN_DESIGN_BRIEF.md`.

## 0. Что это и для кого
**Передарим** (peredarim.ru) — C2C-маркетплейс ресейла подаренных букетов: продавец-физлицо за 2 минуты выставляет полученный букет (фото + размер + свежесть + цена), покупатель берёт свежие цветы дешевле флориста; деньги в эскроу, после сделки — взаимные отзывы; на главной топ «самые свежие» и «самые залайканные». Аудитория — преимущественно женщины 18–40, мобильные, не技-подкованные. **Тон:** тёплый, простой, бережный — «дать букету вторую жизнь», доступная радость и забота, **не** люкс-флорист.

## 1. Платформенный контракт (важно для дизайна)
- Проектируем **один web-интерфейс** (React + Tailwind). Этот же web-build оборачивается Capacitor в iOS/Android — **тот же UI едет на все платформы** (ADR-0004). Отдельных нативных макетов **не нужно**.
- **Mobile-first**: первичный вьюпорт — телефон (360–414 dp). Десктоп — адаптив поверх, не наоборот.
- Превью каждого экрана — в **рамке iPhone/Android**, со скриншотом для pixel-diff baseline.
- Выдаём то, что Claude Design умеет верифицировать: **web-компоненты + дизайн-токены** (не RN).

## 2. Визуальное направление (стартовая гипотеза — уточняется в превью)
- Светлый, воздушный; крупные фото букетов — герой интерфейса, UI вокруг них нейтральный.
- Палитра-направление: спокойный нейтральный фон + один тёплый «живой» акцент + отдельный цвет для «свежести» и для «лайка/сердца». Точные значения — в токенах (§3), не хардкодить в компонентах.
- Скругления мягкие, тени деликатные; ощущение «свежо и аккуратно», без глянца и не «девочковое-клише».

## 3. Дизайн-токены (обязательный артефакт, отдаётся как `tailwind-preset` + CSS-vars)
Определить **семантические** токены (роли, а не сырые цвета), чтобы тема менялась централизованно:
- **Цвет (роли):** `bg`, `surface`, `surface-muted`, `border`, `text`, `text-muted`, `primary` (+hover/active/disabled), `accent`, `success`/`freshness-fresh`, `freshness-aging`, `freshness-old`, `warning`, `danger`, `like` (heart), `on-primary`, `focus-ring`.
- **Типографика:** семейство (системное RU-дружественное), шкала размеров (display/h1/h2/h3/body/caption), веса, line-height; единый компонент `Text`.
- **Spacing:** шкала (4-based: 0/1/2/3/4/6/8/12/16/24…), отступы контейнера, безопасные зоны (notch/home-indicator).
- **Радиусы:** sm/md/lg/pill; **тени/elevation:** 0–3; **z-index** слои (nav/sheet/modal/toast).
- **Breakpoints** (mobile-first): base → sm → md → lg.
- **Motion:** длительности (fast/base/slow) + easings; уважать `prefers-reduced-motion`.
- **Иконки:** один набор (lucide-совместимый), размеры из шкалы.

## 4. Инвентарь компонентов (canon должен экспортировать)
Базовые: `Button` (primary/secondary/ghost/danger; **rest=active по умолчанию**; состояния hover/focus/pressed/loading; disabled только по правилу `INTERACTION_STATES.md §1`), `IconButton`, `Input`, `Textarea`, `Select`, `OtpInput` (6 ячеек), `Checkbox`, `Switch`, `Chip/Tag`, `Badge`, `Avatar`, `Money` (копейки→₽), `Skeleton`, `Spinner`, `Toast`, `Modal`, `BottomSheet`, `Tabs`, `SegmentedControl`.
Доменные: `BouquetCard` (фото-каррусель, `FreshnessBadge`, `LikeButton` со счётчиком, цена, город), `FreshnessIndicator` (today/1-2д/3+), `LikeButton`, `RatingStars` (ввод + чтение), `ReviewItem`, `SellerBadge` (рейтинг + кол-во сделок), `PhotoUploader` (1–5, превью, прогресс), `DealStatusStepper` (created→paid_held→released/disputed), `ChatBubble` + `ChatInput`, `PriceInput`, `SizeSelector` (S/M/L/XL), `CitySelector`, `DeliveryToggle` (самовывоз/курьер), `FeedSectionHeader` («Самые свежие»/«Самые залайканные»), `BottomNav` (4–5 вкладок), `TopAppBar`, `EmptyState`.
Для **каждого** компонента — все состояния: default / hover (web) / pressed / focus / disabled / loading / error / empty.

## 5. Экраны (полный список — в `SCREEN_INDEX.md`)
Для каждого экрана задать: цель, ключевые элементы, **главное действие**, состояния (loading/empty/error/success), мобильный лейаут.
Минимальный набор MVP:
1. **Главная / витрина** — две топ-ленты («самые свежие», «самые залайканные») по выбранному городу, `CitySelector`, поиск/фильтр, `BottomNav`.
2. **Карточка букета** — фото-каррусель, свежесть, лайк, цена, продавец+рейтинг, кнопка «Купить» / «Предложить цену», доставка.
3. **Продать** (публикация) — `PhotoUploader`, `SizeSelector`, `FreshnessIndicator`, `PriceInput`, город/гео, превью, публикация.
4. **Вход** — телефон → `OtpInput`; согласие 152-ФЗ.
5. **Сделка** — `DealStatusStepper`, чат сделки (`ChatBubble`/`ChatInput`), способ доставки/трекинг, кнопка «Подтвердить получение», спор.
6. **Отзыв** — `RatingStars` + текст (после релиза, взаимный).
7. **Профиль продавца** — рейтинг, отзывы, активные объявления.
8. **Уведомления** + **пустые состояния** + **ошибки/нет сети**.
9. **Админ/модерация** (web-десктоп) — очередь `pending_review`, действия approve/reject.

## 6. Состояния, адаптив, доступность
- **Исчерпывающая проработка состояний — обязательна.** Все интерактивные состояния (rest/hover/focus-visible/pressed/selected/loading/disabled/error), все состояния инпутов (включая hover и focus), и весь жизненный цикл коллекций (loading-skeleton / empty / **no-results** / loading-more / end-of-list / error / offline) — по `INTERACTION_STATES.md`. Состояния, зависящие от данных, привязываются к `API_CONTRACT.md`.
- **Кнопки по умолчанию активны** (rest = enabled). `disabled` — только in-flight (анти-дабл-сабмит) или жёсткое предусловие; запрещён паттерн «серая кнопка пока форма невалидна» — кнопка активна, валидация инлайн по нажатию (см. `INTERACTION_STATES.md §1`).
- Поиск может вернуть 0 → отдельное состояние «ничего не найдено» с подсказками (≠ пустой ленты).
- Mobile-first; проверить на узком (360) и широком телефоне, затем десктоп.
- **A11y WCAG 2.1 AA:** контраст ≥ 4.5:1 для текста; видимый focus-ring; цели нажатия ≥ 44×44px; семантика (роли, лейблы) на RU; не передавать смысл только цветом (свежесть — цвет + иконка/текст).

## 7. Копирайтинг
- Весь UI-текст — **RU**, тон из §0. Бренд «Передарим» — **только кириллицей** в customer-facing copy (`rebloom` — нигде в UI).
- Канонические строки ведём в `../COPY.md` (создаётся отдельно); в макетах — реалистичные RU-тексты, не lorem.

## 8. Что отдать и как проверить (Definition of Done для хэндофа)
- Превью каждого экрана/компонента в Claude Design + **скриншот в рамке устройства** (baseline для pixel-diff).
- **Токены** как `tailwind-preset` + CSS-vars.
- Компоненты как **web React + Tailwind**, со всеми состояниями.
- Упаковка и передача — строго по `CANON_PACKAGE_TZ.md` (структура zip, версия, CHANGELOG, baselines).
- Критерий приёмки в коде: `npm run test:visual` в `web/` проходит с diff ≤ 2% против выданных baselines.
