# MOTION & MICRO-INTERACTIONS — ТЗ для Claude Design · Передарим (`rebloom`)

> Современные ховеры и анимации — часть дизайн-системы, а не «украшение поверх». **Требование: моушн реализуется внутри `@rebloom/canon` и доезжает на прод в составе пакета** (не дорисовывается руками в `web/`). Парные: `DESIGN_BRIEF.md`, `INTERACTION_STATES.md`, `CANON_PACKAGE_TZ.md`.

## 1. Принципы (тренды 2026, но со вкусом и под деньги)
- **Микровзаимодействия на каждое действие**: тап/клик дают мгновенную тактильную отдачу (press-scale ~0.97, тень/яркость), лайк — пружинный «поп» сердца + счётчик-роллап.
- **Spring/ease, а не линейность**: естественная физика (spring) для появления листов/модалок/карточек; стандартные ease-out для входа, ease-in для выхода.
- **Shared-element / continuity**: при переходе лента → карточка букета фото «продолжается» (shared element), а не резкая смена экрана.
- **Skeleton-shimmer** вместо спиннеров для загрузки ленты; **optimistic UI** для лайка (мгновенно, откат при ошибке).
- **Scroll-reveal/stagger**: карточки ленты мягко проявляются каскадом при скролле (тонко, без «прыжков» layout).
- **Hover (pointer)**: карточка — лёгкий lift (translateY -2–4px) + усиление тени + плавный масштаб фото внутри рамки; кнопки — смена фона/тени; инпут — анимированный focus-ring и плавающий лейбл. На тач — эквивалент в `pressed`.
- **Haptics на мобиле**: лёгкая вибрация на лайк/успех/ошибку (через Capacitor Haptics) — где уместно.
- **Тосты/статусы**: плавный slide-in + auto-dismiss; смена статуса сделки — анимированный степпер.

## 2. Бюджеты и качество (обязательно)
- Длительности: micro 120–200ms, переходы экранов 250–350ms. Ничего «долгого».
- Только **GPU-friendly** свойства: `transform`/`opacity` (не анимировать layout: width/top/left).
- **60fps**, без jank; не блокировать ввод во время анимации.
- **`prefers-reduced-motion`**: полноценный режим без движения (мгновенные смены, без parallax/stagger) — обязателен (a11y).
- Моушн не мешает деньгам: на платёжных/критичных действиях — сдержанно, без задержек подтверждения.

## 3. Моушн-токены (часть дизайн-системы)
Определить как токены (в `@rebloom/canon/tokens`): `duration.fast/base/slow`, `easing.standard/decelerate/accelerate/spring`, `motion.press-scale`, `motion.card-hover-lift`, `motion.stagger-step`. Компоненты используют токены, не магические числа.

## 4. Технический контракт экспорта в canon → прод (важно для вашего «чтобы доезжало»)
- Анимации реализуются **внутри компонентов canon**: CSS-переходы/`@keyframes` для простого; для сложного (spring, shared-element, stagger) — анимационная библиотека, импортируемая внутри пакета (например, Framer Motion), как зависимость canon.
- Моушн-токены и хелперы (варианты Motion, easing) экспортируются из `@rebloom/canon` (entry `./motion`).
- `web/` **не дописывает анимации руками** — он получает их вместе с компонентами при вендоринге пакета. Любая правка анимации = новая версия canon (round-trip), а не локальный патч (`CANON_PACKAGE_TZ.md §10`).
- В `package.json` canon: Framer Motion (или эквивалент) — в `dependencies`/`peerDependencies` по факту реализации; reduced-motion встроен в компоненты.

## 5. Верификация (моушн ≠ статичный скриншот)
- Pixel-diff проверяет **конечные кадры** состояний (rest/hover/pressed/focus) — baseline на каждое (см. INTERACTION_STATES).
- Моушн дополнительно проверяется **чеклистом** + при возможности коротким видео/демо взаимодействия из Claude Design (превью). На проде — ручной smoke по чеклисту §1–§2 + проверка reduced-motion.
- DoD моушна: токены заданы; hover/pressed/focus анимированы; лайк/skeleton/переходы реализованы в canon; reduced-motion работает; 60fps; всё пришло пакетом, ничего не дорисовано в `web/`.
