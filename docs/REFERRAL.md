# REFERRAL — Передарим (`rebloom`) · post-MVP, feature-flagged

> Реферальная программа для роста; влияет на атрибуцию (analytics). По умолчанию **выключена** флагом; включаем после стабилизации экономики (известна комиссия/юнит-экономика).

## 1. Модель (черновик)
- У пользователя — персональная ссылка/код. Приглашённый регистрируется по коду → атрибуция `attribution.referrer_id` (first-touch, не перебивает UTM-источник, а дополняет).
- Награда — на условиях оператора (напр. бонус/снижение комиссии после первой успешной сделки приглашённого). Конкретику задаёт оператор `[verify]`.

## 2. Антифрод (обязательно)
Реферальные награды — частая цель накрутки: проверять self-referral (тот же IP/устройство/карта), мульти-аккаунты, фейковые сделки (см. ADMIN_BACKEND_TZ §Fraud). Награда — только после **подтверждённой** сделки, не за регистрацию.

## 3. Бэкенд
`core/referral` (за флагом), поля атрибуции, защита от накрутки, отчёт в админке (источник = referral). Награды через ledger/комиссию.

## 4. Задачи
Epic E12 (T12.6 referral за feature-flag + анти-накрутка). Не входит в первый релиз.
